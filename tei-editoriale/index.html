Bienvenue!

Ce site a été réalisé dans le cadre du cours «Documents Structurés» du M2.

«PluriTAL»

L’objectif de ce projet est de réaliser une chaîne de traitement pour la publication de documents en ligne.

À partir des documents fournis, nous avons crée ce site web pour présenter le contenu de ces documents et interagir avec.

Organisation du site

Deux grandes étapes ont été mises en œuvre.

Partie 1.

La première étape consiste à transformer les documents au format OpenDocument en document XML selon le standard de la Text Encoding Initiative :

«TEI»

Les trois documents traités sont «Madame-Bovary.odt», «SIDDHARTHA.odt» et «Wuthering-Heights.odt».

En effet, le format .odt est une archive contenant les différentes instructions pour interprétation par l'éditeur.

Après avoir dézippé les archives de plusieurs fichiers XML, nous les avons explorées pour extraire le contenu du document et ses métadonnées.

Nous avons employé XSLT pour réaliser cette extraction. Puis, un schéma RelaxNG nous a permis de valider la structure.

Partie 2.

Une fois la première étape terminée, les documents sont stockés dans une base de données XML :

«eXist-db»

À partir de celle-ci, nous avons réalisé un site internet à l’aide de xquery et xslt.

Navigation

Sur ce site nous présentons trois interfaces :

une page d'accueil pour la présentation du projet,

une page pour la recherche d’informations dans les documents :

«Recherche»

et une autre pour afficher les documents mis en page :

«Visualisation»

Les difficultés rencontrées

Partie 1 : Au stade de l'extraction, comme nous ne connaissons pas très bien la structure TEI, il a fallu plus de temps que prévu pour terminer la tâche.

Une fois que nous avons pu extraire les informations nécessaires, nous n'avons pas pu les valider immédiatement en raison du fichier RelaxNG incorrect.

Cela a été immédiatement résolu dès que le fichier RelaxNG a été corrigé.

Partie 2 : Lorsque de la création de la page de présentation, nous voulions insérer des liens dans les textes comme nous le faisons en HTML.

Cependant, la grammaire RelaxNG nous permet pas de mettre un element <LIEN> au milieu des <text/> dans le fichier XML.

C'est pour cette raison que les liens sont placés sur une ligne séparée et non intégrés dans les phrases.

Concernant eXist-db, c'est une toute nouvelle plateforme pour nous. Parfois, la manipulation était déroutante et nous devions rechercher des solutions en ligne.

L'écriture de xQuery pour les fonctions de recherche était également difficile.

Comme nous n'avions pas beaucoup de temps, nous avons décidé de faire une simple visualisation des documents sans utiliser de CSS.
